# Glossary

Key terms used throughout this book.

| Term | Definition |
|------|-----------|
| **Physical AI** | AI systems that operate in and interact with the physical world through sensors and actuators |
| **Embodied Intelligence** | Intelligence that emerges from the interaction between an agent's body and its environment |
| **ROS 2** | Robot Operating System 2 — middleware for robot software development |
| **Humble Hawksbill** | The LTS release of ROS 2 targeting Ubuntu 22.04 |
| **URDF** | Unified Robot Description Format — XML format for describing robot geometry |
| **Xacro** | XML macro language for generating URDF files with parameters |
| **Gazebo** | Physics simulation engine for robotics |
| **Fortress** | The LTS release of Gazebo compatible with ROS 2 Humble |
| **Digital Twin** | A virtual replica of a physical system used for simulation and testing |
| **NVIDIA Isaac Sim** | NVIDIA's robotics simulation platform built on Omniverse |
| **Omniverse** | NVIDIA's platform for 3D simulation and collaboration |
| **VSLAM** | Visual Simultaneous Localization and Mapping |
| **Nav2** | Navigation 2 — the ROS 2 navigation framework |
| **Costmap** | A grid-based representation of obstacles used by Nav2 for path planning |
| **VLA** | Vision-Language-Action — architecture connecting perception, language, and motor control |
| **Whisper** | OpenAI's speech-to-text model |
| **LLM** | Large Language Model — used for intent parsing and task planning |
| **Sim-to-Real** | Transfer of behaviors from simulation to physical hardware |
| **Domain Randomization** | Varying simulation parameters to improve sim-to-real transfer |
| **JetPack** | NVIDIA's SDK for Jetson edge computing platforms |
| **Jetson Orin** | NVIDIA's edge AI computing module for robotics |
| **DDS** | Data Distribution Service — the communication middleware used by ROS 2 |
| **Action Server** | A ROS 2 pattern for long-running tasks with feedback and cancellation |
| **Behavior Tree** | A hierarchical control structure used by Nav2 for navigation decisions |
| **Safety Filter** | A validation layer between LLM output and robot actuation |
